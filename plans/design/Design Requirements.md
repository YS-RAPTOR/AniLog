# Design Requirements

Mainly four groups of pages are needed for the application.

- Login
- Onboarding
- app/anime
- app/manga (include light novels)

Each of these groups of pages should have designs that work well in mobile portrait, mobile landscape, tablet portrait, tablet landscape, and desktop. The designs should be consistent across all platforms with layout changes to accommodate the different screen sizes and orientations. The design should be clean, modern, and user-friendly, with a focus on ease of navigation and readability. The color scheme should be visually appealing and consistent throughout the application. Additionally, the design should incorporate accessibility features to ensure that it is usable by people with disabilities.

# app/anime and app/manga

The app is primarily focused on anime and manga, so the design should reflect this theme. The use of anime and manga-inspired graphics, icons, and typography can help to create a cohesive and engaging user experience. The design should also prioritize the display of anime and manga content, with clear and visually appealing layouts for browsing and discovering new titles. Overall, the design should be tailored to the interests and preferences of anime and manga fans while also providing a seamless and enjoyable user experience across all platforms.

The primary purpose of the application is to provide a better user experience on top of myanimelist.net. The design should focus on improving the user experience by providing a more intuitive and visually appealing interface for browsing and managing anime and manga lists. The design should also prioritize the display of anime and manga content, with clear and visually appealing layouts for browsing and discovering new titles.

The main improvement over the existing myanimelist website will be the user interface and user experience. The design should focus on creating a more modern and visually appealing interface that is easier to navigate and use.

One of the biggest improvements will be that within the app/manga and app/anime the fundamental building block will be the list. A list is a collection of anime/manga. A list will have to support the following features:

- Each entry will have three views that it can switch:
    - List view: Shows each entry as a row within a table, with different columns indicating different information about the entry.
    - Card view: Each entry is represented mainly by its image, with extra customizable information shown below the image.
- Each of the different views will have extra customization options, for example the columns shown in the list view are fully customizable. Same for the information shown in the card view. Both List View and the Card View should be able to show the same information, just in different formats.
- Each list can have a grouping associated with it. Multiple keys can be used for grouping, for example the status of the entry (watching, completed, etc), the score given to the entry, the format of the entry (tv, movie, ova, etc), and many more. The user should be able to choose which keys they want to use for grouping and how they want to group them. For fields that are not categorical, for example the score, the user should be able to choose how they want to categorize them (ranges of scores, for example 0-3, 4-6, 7-8, 9-10). For groups keys like genres where an entry can have multiple values, each entry will show up in each of the groups it belongs to. For example if an anime has the genres action and comedy, it will show up in both the action and comedy groups. The user should be able to choose the order of the groups using a drag and drop interface and also quick sort the groups by different criteria (alphabetically, by number of entries, etc). The user should also be able to collapse and expand the groups to show or hide the entries within them.
- The sorting of the entries can be global and be overridden at the group level. For example, globally the entries can be sorted by title, but within a specific group they can be sorted by score.
- The final is I want a very powerful filtering system that allows the user to filter the entries based on any of the fields that are tracked for each entry. For example, the user can filter the anime list to only show anime with a score of 8 or higher, or only show manga that are in the "completed" status. The filtering system should also support multiple filters at the same time, for example the user can filter the anime list to only show anime with a score of 8 or higher and that are in the "watching" status. The filtering system should be easily accessible and intuitive to use, with clear options for selecting the fields to filter by and the values to filter for. The filtering system should support both and, or, and not logic for combining multiple filters, allowing the user to create complex filters that suit their needs. The filtering system should also provide real-time feedback on the number of entries that match the current filter criteria, so that the user can easily see the impact of their filter choices and adjust them as needed. The filtering system should also allow comparing values between entry data. For example, the user can filter the anime list to only show anime that have a higher score than another anime, or only show manga that have a chapter count that is greater than the currently read chapter count. The filter system should also support some simple math operations for numeric fields, for example the user can filter the anime list to only show anime that have a score that is at least 2 points higher than the score of another anime. This allows the user to easily find entries that are similar or different to other entries based on the values of their fields, and also allows them to easily find entries that meet specific criteria based on comparisons between their fields.
- The user should be able to track values for each entry and do actions when those values change. For example, the user can what is the latest episode that has beein released for an anime, and when that value changes the user can choose to be notified about it. Another example is the user can track the score they have given to an anime, and when that value changes they can choose to be notified about it.
- The user should also be able to export and import all of the settings for the list, either by copying the json to the clipboard or by downloading/uploading a file. This includes the entries in the list, the grouping settings, the sorting settings, and any other customization options. This allows the user to easily back up their lists and settings, or to share them with others. This can be implemented in the settings screen. All of the settings will also be stored in the users google drive account so that the lists and settings are synced across all of the users devices.
- The user should also be able to export the lists into different formats (csv, json, xml, etc) and also export them into separate functions.
    - Some functions I want to implement are:
        - Exporting to nyaa.si magnet links for anime and manga entries. This allows the user to easily access the content for each entry directly from the app, and also share those links with others.
        - Exporting to youtube music playlists for anime entries. This allows the user to easily access the music for each entry directly from the app, and also share those playlists with others.
- In the global settings I want to be able to setup entry links to other websites. For example, I want to support nyaa.si torrent links for anime and manga entries, so that when the user clicks on the link for an entry it will take them to the nyaa.si page for that entry. The user should be able to customize which websites they want to have links for and also customize the url format for each website. For example, the url format for nyaa.si could be https://nyaa.si/?f=0&c=0_0&q={title} where {title} is replaced with the title of the entry. This allows the user to easily access additional information and resources for each entry directly from the app. I want any field that is tracked for an entry to be able to be used in the url format, for example if the user is tracking the score they have given to an anime, they could use that score in the url format to link to a page that shows anime with similar scores. Similar thing for the latest episode released, the user could link to a page that shows the airing schedule for that episode. The possibilities are endless and it allows the user to customize their experience and access the information they care about in a way that is convenient for them. This links field can be then displayed in the entry within the lists. The application will automatically hit the url and get the favicon for the website and show it next to the link for easy identification. The user can also choose to show the name of the website next to the favicon, or just show the favicon without the name. This will then be afield that can be accessible in the list view and card view and will support all the same customization options as the other fields, for example the user can choose to show it in the list view but not in the card view, or they can choose to show it in both views but only for certain groups. The user can also choose to open the links in a new tab or in the same tab when they click on them. This allows the user to easily access additional information and resources for each entry directly from the app, and customize their experience to fit their needs and preferences.
- The user should be able to have multiple lists, for example a list for anime and a list for manga. Each list can have its own settings for grouping, sorting, and customization options. The user should be able to easily switch between lists and manage them from a central location in the app. This allows the user to keep their anime and manga lists separate and organized, while still being able to access all of their content from one place. The user should also be able to create custom lists for specific purposes, for example a list of anime they want to watch in the future, or a list of manga they have read and want to recommend to others. This allows the user to further customize their experience and organize their content in a way that makes sense to them.

# login

When the user clicks on it, they will be taken to the mal login page where they can enter their credentials and authorize the app to access their account. After they authorize the app, they will be redirected back to the app and logged in. The design of this page should be simple and straightforward, with a clear call to action for the user to log in with their mal account. The design should also be visually appealing and consistent with the overall theme of the app.

A similar login page will be added for google login, where the user can log in with their google account instead of their mal account. This will allow users who do not have a mal account to still use the app and access its features. The design of this page should also be simple and straightforward, with a clear call to action for the user to log in with their google account. The design should also be visually appealing and consistent with the overall theme of the app.

# Onboarding

There are three main onboarding pages that are needed for the application.

- Beginner: This will have step by step instructions on how to create the necessary myanimelist client id with the necessary redirect urls, app settings and all of that, the google client id with the necessary scopes and redirect uris. This will be a truly hands on onboarding experience where the user will have to follow the instructions and input the necessary information in order to use the app. This onboarding experience is for users who are not familiar with creating client ids and setting up apps on myanimelist and google, and need a lot of guidance to do so.
- Advanced: This will have the same instructions as the beginner onboarding, but it will also have explanations for each step and why it is necessary. This onboarding experience is for users who are familiar with creating client ids and setting up apps on myanimelist and google, but still want some guidance and explanations to do so. Only the main information like the needed redirect urls, app types, and scopes will be shown, but the user will have to know how to use that information to create the client ids and set up the apps on their own.
- Import: This onboarding experience is for users who already have the necessary client ids and app settings set up, either from using a previous version of the app or from following the beginner or advanced onboarding. This onboarding will simply have a text field that expects a json with all the necessary information for the app to work. This allows users to easily import their settings and get started with the app without having to go through the onboarding process again. The design of this page should be simple and straightforward, with a clear call to action for the user to input their json and import their settings. The design should also be visually appealing and consistent with the overall theme of the app. The exported json can be obtained from the settings page within the app, where the user can choose to export secrets. This will copy the json to the clipboard.
